# Memory Rules

These are the rules that make memory work:

---

## Session Start
- Read USER.md → know who the human is
- Read SOUL.md → know how to act
- Read HEARTBEAT.md → know what to check
- Read last 2-3 daily notes → know what's been happening

---

## During Conversation
- When human says "Remember this" → write to today's daily note
- When human says "don't forget" → write to memory immediately
- Update memory after important conversations
- Log decisions and why they were made

---

## Session End
- Write summary to `memory/daily/YYYY-MM-DD.md`
- Include: what was done, decisions, next steps
- Note any "worth keeping" info for MEMORY.md

---

## Memory Maintenance (weekly)
- Review daily notes
- Update MEMORY.md with distilled important info
- Clean up old or outdated notes
- Keep MEMORY.md light (just pointers, not full data)

---

## The Golden Rules

1. **If it's worth remembering, write it down**
2. **No "mental notes" — files survive restarts**
3. **Memory is built in drops, lost in buckets** — be consistent
4. **Update MEMORY.md after every significant session**
